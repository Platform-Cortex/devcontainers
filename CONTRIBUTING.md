## 🚀 Getting Started

### 1. Reporting Bugs & Suggesting Features

Before opening a new Issue, please search our existing Issues or Discussions to ensure the topic hasn't been covered yet.

When opening an Issue, use the appropriate template and provide as much detail as possible:

* Clear steps to reproduce the issue.
* Expected behavior vs. actual behavior.
* Relevant runtime logs, environment configuration, and software versions.

### 2. Pull Request Workflow

1. **Fork** the corresponding repository.
2. Create a feature branch for your changes: `git checkout -b feature/your-feature-name` or `git checkout -b fix/bug-fix-name`.
3. Ensure unit and integration tests are written for your changes.
4. Verify that all linters, formatting checks, and local CI workflows pass completely.
5. Commit your changes following the **Conventional Commits** standard:
   * `feat: add adaptive remediation agent to AIOps engine`
   * `fix: resolve memory leak in database connection pool`
6. Push changes to your fork and open a **Pull Request** targeting the `main` branch.

## 🛠️ Code Quality Standards

Each specific repository inside the organization runs automated checks (Linters, Formatters, SonarQube, etc.). We request extra diligence regarding test coverage for any code altering the core loops of our autonomous intelligent agents (Agentic AI) or streaming data pipelines (AIOps).

## 💬 Need Help?

If you have questions about our architectural decisions or need guidance on how to contribute, please feel free to open a new thread in the **Discussions** tab of the `.github` repository.
